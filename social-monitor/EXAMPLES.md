## 📋 示例 2: 竞品分析

### 场景
产品经理想了解竞品在知乎和小红书上的用户评价。

### 使用步骤

**1. 监控多个竞品**
```
# 竞品 A
/social-monitor 竞品A 知乎 3天

# 竞品 B
/social-monitor 竞品B 知乎 3天

# 竞品 C
/social-monitor 竞品C 小红书 3天
```

**2. 对比分析**

将三次监控结果保存下来，进行对比分析：

```markdown
### 竞品对比分析

| 竞品 | 内容数 | 平均点赞 | 主要话题 | 用户态度 |
|------|--------|----------|----------|----------|
| 竞品A | 45条 | 230 | 功能强大、价格贵 | 正面 70% |
| 竞品B | 32条 | 180 | 性价比高、bug多 | 中性 60% |
| 竞品C | 28条 | 150 | 界面好看、功能少 | 正面 55% |

**洞察**:
- 竞品A功能领先但价格偏高，存在机会
- 竞品B性价比是优势，但稳定性问题明显
- 竞品C主打设计，功能深度不足
```

## 📋 示例 3: 危机预警

## 🔧 进阶技巧

### 技巧 1: 组合关键词

```
# 监控品牌+特定话题
/social-monitor "品牌名 质量" 小红书 1天
/social-monitor "品牌名 客服" 微博 1天
/social-monitor "品牌名 vs 竞品" 知乎 3天
```

### 技巧 2: 多平台对比

```
# 同一关键词在不同平台的表现
/social-monitor AI 小红书 1天  # 生活化、体验向
/social-monitor AI 知乎 1天    # 深度、专业向
/social-monitor AI 微博 1天    # 新闻、热点向
```

### 技巧 3: 时间窗口分析

```
# 对比不同时间窗口的数据
/social-monitor 品牌名 小红书 6小时  # 实时热度
/social-monitor 品牌名 小红书 1天    # 日度趋势
/social-monitor 品牌名 小红书 7天    # 周度趋势
```

### 技巧 4: 定向监控 KOL

```
# 监控特定账号（如果平台支持）
/social-monitor @某KOL 小红书 1天
/social-monitor @竞品官方账号 微博 3天
```

## 📝 最佳实践总结

### ✅ DO - 推荐做法

1. **先手动测试，再设定定时**
   - 确保关键词、平台、时间范围配置正确

2. **控制监控频率**
   - 品牌监控: 每天 1-2 次
   - 危机预警: 每 6-12 小时
   - 行业追踪: 每天 1 次

3. **保存历史数据**
   - 用于趋势对比分析
   - 生成周报、月报

4. **结合人工判断**
   - AI 抓取数据，人工判断重要性
   - 关键负面内容及时响应

### ❌ DON'T - 避免做法

1. **过高频率抓取**
   - ❌ 每小时抓一次
   - ✅ 每天 1-2 次即可

2. **监控过于宽泛的关键词**
   - ❌ "AI"（结果太多且不相关）
   - ✅ "品牌名+AI"（更精准）

3. **忽视账号风险**
   - ❌ 用主账号高频抓取
   - ✅ 使用测试账号或降低频率

4. **过度依赖自动化**
   - ❌ 完全依赖监控报告
   - ✅ 重要内容人工复核

## 🆘 故障处理示例

### 场景: 抓取失败

```
❌ 错误: 小红书抓取失败，返回空结果

解决步骤:
1. 检查 web-access
   $ node ~/.claude/skills/web-access/scripts/check-deps.mjs

2. 测试平台连通性
   /social-monitor test 小红书

3. 检查是否需要登录
   - 在 Chrome 中手动访问小红书
   - 确认是否被要求登录
   - 登录后重试

4. 如果仍失败，查看详细错误
   - 检查 CDP Proxy 日志
   - 可能是选择器失效，需要更新
```

# Social Monitor - 社交媒体内容监控工具

> 基于 web-access 构建的智能社交媒体监控 skill

## 🎯 功能特性

- ✅ **多平台支持**: 小红书、微博、知乎
- ✅ **定时监控**: 自动定期抓取最新内容
- ✅ **智能过滤**: 按时间范围筛选内容
- ✅ **结构化报告**: 清晰的数据呈现
- ✅ **去重机制**: 避免重复内容
- ✅ **历史记录**: 可选保存历史数据

## 🚀 快速开始

### 最简单的用法（默认监控爱奇艺）

```bash
# 直接使用，无需参数！
/social-monitor
```
**默认行为**: 监控小红书上最近1天关于"爱奇艺"的内容

### 自定义用法

```bash
# 监控微博上的爱奇艺
/social-monitor 爱奇艺 微博 1天

# 监控其他关键词
/social-monitor 人工智能 微博 12小时
/social-monitor Claude 知乎 3天
```

### 设置定时任务

```bash
# 1. 进入配置向导
/social-monitor setup

# 2. 按提示配置监控任务
关键词: 你的品牌名
平台: 小红书,微博
执行时间: 每天 09:00
时间范围: 24小时

# 3. 确认配置
任务会自动在指定时间执行并推送结果
```

### 管理任务

```bash
# 查看所有监控任务
/social-monitor list

# 停止指定任务
/social-monitor stop task_001

# 测试平台连通性
/social-monitor test 小红书
```

## ⚠️ 注意事项

### 账号安全

- ⚠️ 频繁自动化操作可能触发平台风控
- ✅ 建议使用测试账号
- ✅ 控制监控频率（每天 1-2 次为宜）
- ✅ 添加随机延迟

### 数据合规

- ⚠️ 仅用于个人研究、品牌监控等合法用途
- ⚠️ 不要用于商业化爬虫
- ⚠️ 遵守各平台的服务条款
- ⚠️ 注意保护用户隐私

### 技术限制

- 小红书强反爬，必须使用 CDP 模式
- 部分内容可能需要登录才能查看
- DOM 结构变化可能导致选择器失效（需更新）

## 🔧 故障排查

### 问题 1: 抓取失败

**现象**: 返回空结果或错误
**解决**:
```bash
# 1. 检查 web-access 是否正常
node ~/.claude/skills/web-access/scripts/check-deps.mjs

# 2. 检查 CDP Proxy
curl http://localhost:3456/health

# 3. 测试平台连通性
/social-monitor test 小红书
```

### 问题 2: 定时任务未执行

**现象**: 到了设定时间但没有执行
**解决**:
```bash
# 1. 检查任务配置
/social-monitor list

# 2. 检查 cron 任务
# 定时任务仅在 Claude Code 会话活跃时执行
```

### 问题 3: 账号被封禁

**现象**: 平台提示账号异常
**解决**:
- 降低监控频率
- 使用独立测试账号
- 添加随机延迟
- 暂停一段时间后再尝试
